# 데이터 타입

ES6 는 7개의 데이터 타입을 제시한다

- 숫자
- 문자열
- 불리언
- undefined
- null
- 심벌
- 객체 - 객체, 함수, 배열



## 숫자타입

자바스크립트의 특징이라면 숫자가 '실수','정수' 이런식으로 나눠지지 않는다 라는 점이다.

**모두~~~~ 실수다.**

정수로 보이는가? 그래도 결국 실수로 처리된다.

예를 들어 `0b1001101` 이런것은 결국 출력이 될때 실수값으로 나오게 된다.

그외에는

- `Infinity` : 양의 무한대
- `-Infinity` : 음의 무한대
- `NaN` : 산술연산 불가
  - 자바스크립트는 대소문자를 구별한다. 
  - `Nan` `nan` 이런식으로 쓰지말자.



## 문자열 타입

문자열은 `""` , `''` , 백틱(`) 들을 사용해 감싸면 그게 문자열이다.

가장 일반적인것은 작은따옴표(`''`)

이 '따옴표'로 감싸지 않으면 자바스크립트 엔진이 키워드나 토큰으로 인식한다.

자바스크립트의 문자열은 **원시타입**이며 **변경불가능한 값**이다.



### 템플릿 리터럴

ES6 부터 **템플릿 리터럴** 이라고 하는 표기법이 도입되었다.

이거는 백틱을 써야한다.

템플릿 리터럴을 통해서 할수있는 것은

- 멀티라인 문자열

  - 줄바꿈이나 공백이 그냥 인식된다. 
  - 원래는 "이스케이프 시퀀스"라고 `\n` 이런걸 달아줘야 줄바꿈이 되었다.

- 표현식 삽입

  - `${}` 을 사용해서 간단하게 표현식을 문자열에 넣을 수 있게 되었다.

  - 원래의 방식(ES5)

    - ```js
      var name="shin";
      console.log('저의 성씨는'+name+'입니다.');
      ```

  - 개선된 방식(ES6)

    - ```js
      var name="shin";
      console.log(`저의 성씨는 ${name} 입니다.`);
      ```

- 태그드 템플릿

- 등등이 있다.

## undefined 타입

이는 변수가 `undefined`로 초기화하는데 사용하는 값이다.

의도적으로  `undefined` 로 설정하는 것은 옳지않는 방법 

값이 없는것을 보여주기 위해서는 `null`을 쓰자



## null 타입

값이 없다는 것을 의도적으로 보여주는 타입

이는 값을 아무것도 주지않겠다는것은  해당 메모리에 가비지 콜렉션이 수행될것이다.



함수가 유효한 값을 반환할 수 없는 경우 `null`을 반환하기도 한다.



## 심벌

ES6 에서 새로 추가된 원시타입 값으로 

심벌 값은 다른 값과 중복되지 않는 유일무이한 값이다.

이름이 충돌할리가 없는 객체의 유일한 프로퍼티 키를 만들기 위해서 사용된다.

## 데이터 타입이 왜필요한가

왜냐면 값을 메모리에 저장하려면 일정 공간을 확보해야하기 때문

자바스크립트 엔진은 타입에 따라 정해진 크기의 메모리 공간을 확보한다.



정리하면

- 값을 저장할때 메모리 공간의 크기르 결정하기위해
- 값을 참조할때 한번에 읽어야할 메모리 공간의 크기를 결정하기 위해
- 메모리에서 읽은 2진수를 어덯게 해석할지 결정하기 위해



## 동적타이핑



C나 자바의 경우 정적 타입언어다.

이는 변수를 선언하면 사전에 데이터 타입을 선언해야한다. 이를 "명시적 타입 선언" 이라고 한다.

정적타입언어는 컴파일 시점에서 타입체크를 하고 실행을 진행하거나 막는다.

좀더 안정적이고 런타임 에러를 줄인다.



자바스크립트는 동적타입언어라 변수를 선언시 타입을 선언하지않는다.

그래서 데이터 값을 자유롭에 할당이 가능하다.



좀더 생각보면 할당을 통해서 타입 결정된다는 것이다. (타입추론)

그리고 재할당을 통해서 언제든지 동적으로 변할수있다.



이를 "동적 타이핑"이라고 한다.



장점도 있겠지만 단점도있다.

- 변화하는 변수값을 추적하기가 어렵다.
- 변수 확인전 까지는 타입을 확신이 불가능하다.
- 즉, 유연성은 높지만 신뢰성은 떨어지는것이다.



이처럼 안정성을 높이기위해서 변수를 사용시 주의사항이 있다.

- 변수는 필요한 경우에만 쓰자
- 스코프를 최대한 좁게만들자
- 전역변수는 최대한 쓰지말자
- 변수보다 상수를 써서 값의 변경을 억제하자
- 가독성높게 네이밍 하자
